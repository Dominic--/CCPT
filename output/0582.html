>>>> 下面是打印输出信息：
>>>> Hello world!!.
>>>> I am process 2.
>>>> hello pass.
>>>> listproc: head=c01a81b0,pid=-1-->le=c05ea1e8,pid=2-->le=c05ea168,pid=1
>>>> do_wait return 0
>>>> listproc: head=c01a81b0,pid=-1-->le=c05ea168,pid=1
>>>> all user-mode processes have quit.nr_process=2
>>>> init check memory pass.
>>>> kernel panic at kern/process/proc.c:479:
>>>> initproc exit.
>>>
>>>完全正常。kernel认为在任何时候都应该至少存在一个用户态进程（内核线程不算），目前uCore的做法是在所有用户态进程退出之后进行一些内存检查（主要用来检测内存泄露），然后直接panic，
>> 至少目前不意味着任何错误。
